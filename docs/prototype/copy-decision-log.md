# 文案与双语术语决策记录（MVP）

## 1. 决策目标

统一中英文关键术语，避免页面、邮件、后台状态表达不一致。

## 2. 术语映射（首版）

| 中文 | English | 说明 |
|---|---|---|
| 售后申请 | Support Request | 用户发起入口文案 |
| 工单 | Ticket | 全系统统一术语 |
| 工单号 | Ticket Number | 查询与通知使用 |
| 已提交 | Submitted | 初始状态 |
| 已受理 | Accepted | 客服已接单 |
| 处理中 | In Progress | 处理中状态 |
| 已解决 | Resolved | 处理完成 |
| 已关闭 | Closed | 终态 |
| 问题类型 | Issue Type | 表单字段 |
| 问题描述 | Issue Description | 表单字段 |
| 查询工单 | Track Ticket | 导航与查询页 |
| 提交成功 | Request Submitted | 成功页标题 |

## 3. 文案风格规范

- 英文默认面向美国用户，语气简洁、直接、礼貌
- 中文使用简体，避免过于口语化
- 错误提示遵循“原因 + 可执行建议”格式
  - 示例：`图片格式不支持，请上传 jpg、png 或 webp 文件。`

## 4. 待确认项

- 是否使用“Support Case”替代“Ticket”（当前建议不替换）
- 邮件中称呼格式：`Hi {firstName}` 还是 `Hello`
- 售后政策摘要长度（短版 3 条或长版 5 条）

## 5. 版本记录

- v1.0：建立 MVP 术语基线，供前台、后台、邮件模板统一使用
