# 宠物用品售后网站 MVP 用户流程

## 1. 用户流程 A：提交售后申请（前台）

### 目标
用户在 3 分钟内完成问题提交并获得工单号。

### 主流程
1. 用户进入首页或商品详情页
2. 点击“申请售后”
3. 进入售后表单页，填写必填项
4. 可选上传图片凭证（最多 3 张）
5. 点击提交
6. 系统校验成功，生成工单号
7. 跳转成功页并展示工单号
8. 系统发送确认邮件

### 异常流程
- 必填项缺失 -> 表单就地报错
- 图片格式不支持/超过大小限制 -> 上传失败提示
- 服务异常 -> 提示“提交失败，请稍后重试”

### 成功判定
- 页面展示有效工单号
- 邮件发送成功（异步可重试）

---

## 2. 用户流程 B：查询工单状态（前台）

### 目标
用户能通过工单号和邮箱自助查询进度，减少人工咨询。

### 主流程
1. 用户进入“工单查询”页面
2. 输入工单号和邮箱
3. 点击查询
4. 系统校验并返回工单状态
5. 用户查看状态、更新时间和最新回复

### 异常流程
- 工单号或邮箱格式错误 -> 输入提示
- 工单不存在 -> 友好提示并引导重新检查
- 邮箱与工单不匹配 -> 安全提示，不泄露详情

### 成功判定
- 查询结果在 2 秒内返回
- 状态和更新时间与后台一致

---

## 3. 用户流程 C：客服处理工单（后台）

### 目标
客服可快速完成工单分拣、处理、回复与关闭。

### 主流程
1. 客服登录后台
2. 在工单列表按状态/日期筛选
3. 打开工单详情查看用户描述与附件
4. 更新状态（已受理 -> 处理中 -> 已解决/已关闭）
5. 填写对外回复内容并保存
6. 系统记录操作日志并发送状态变更邮件

### 异常流程
- 工单已被他人更新 -> 提示刷新后重试
- 邮件发送失败 -> 记录重试任务，不阻断状态更新

### 成功判定
- 状态流转完整可追溯
- 对外回复可在前台查询页看到最新结果

---

## 4. 关键状态定义

- `已提交`：用户提交成功，待客服处理
- `已受理`：客服已接单
- `处理中`：已进入问题处理阶段
- `已解决`：处理完成并回复用户
- `已关闭`：工单终态（无需继续处理）
