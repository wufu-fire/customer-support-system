# 宠物用品售后服务网站 MVP 需求文档（PRD）

## 1. 项目概述

- 产品名称：宠物用品售后服务网站
- 目标市场：美国
- 支持语言：英语、简体中文
- 核心能力：
  - 商品展示（不含下单与支付）
  - 售后问题提交与处理

## 2. 问题陈述

当前售后请求分散在多个渠道，导致响应慢、处理口径不一致。需要建设一个集中式、双语化的售后网站，用于统一用户问题受理与工单处理流程。

## 3. MVP 目标

1. 用户可在 3 分钟内完成售后请求提交
2. 客服团队可进行工单分拣并更新状态
3. 用户可通过工单号和邮箱查询处理进度
4. MVP 全部页面支持中英文切换

## 4. 范围定义

### 4.1 本期范围（In Scope）

- 商品列表页与商品详情页
- 带图片上传的售后申请表单
- 工单状态查询页
- 内部后台工单列表/详情/状态更新
- 邮件通知（提交成功通知 + 状态变更通知）
- 关键页面与模板的中英双语支持

### 4.2 非本期范围（Out of Scope）

- 在线购买与支付
- 复杂账号体系
- 在线实时客服
- 自动退款系统集成
- 高级数据分析看板

## 5. 用户角色

- 客户（网站访客）
- 客服人员（内部后台用户）

## 6. 功能需求

### 6.1 商品展示

- 按分类浏览商品列表
- 查看商品详情（图片、规格、使用说明、售后政策摘要）
- 支持基础关键词搜索

### 6.2 售后申请提交

必填字段：
- 姓名
- 邮箱
- 商品名称（自由输入，或从商品列表选择）
- 问题类型
- 问题描述

选填字段：
- 手机号
- 订单参考号

附件要求：
- 最多上传 3 张图片
- 支持格式：jpg、jpeg、png、webp
- 单张图片大小上限默认 5MB（可配置）

系统行为：
- 自动生成唯一工单号
- 返回包含工单号的提交成功页
- 发送确认邮件

### 6.3 工单查询

- 输入：工单号 + 邮箱
- 输出：当前状态、最近更新时间、最近一次对外回复
- 安全要求：
  - 查询失败统一提示，不区分“工单不存在”或“邮箱不匹配”
  - 接口需具备基础限流能力（防暴力枚举）

### 6.4 后台工单处理

- 工单列表支持筛选（状态、日期、问题类型）
- 工单详情可查看完整上下文与附件
- 状态流转：
  - 已提交
  - 已受理
  - 处理中
  - 已解决
  - 已关闭
- 支持添加内部备注和面向客户的回复
- 状态变更触发邮件通知
- 后台最小鉴权基线：
  - 客服人员必须登录后访问后台
  - MVP 采用单角色最小权限（`agent`），后续可扩展为多角色（如 `admin`）
  - 会话超时与失效策略需在实现中明确（JWT 或 Session 二选一）

### 6.5 国际化（i18n）

- 页面头部提供语言切换
- 默认语言为英语
- MVP 页面、表单、错误提示、邮件模板全部提供完整翻译

## 7. 非功能需求

- 页面性能目标：美国常规网络条件下，核心页面加载时间 < 3 秒
- 响应式设计，兼容桌面端和移动端
- 基础可访问性要求（键盘可操作、表单标签清晰）
- 基础安全能力：参数校验、上传约束、防垃圾提交策略
- 提供隐私政策与法律声明页面

## 8. 验收标准

1. 用户可浏览商品内容，且无购买流程
2. 售后提交可成功生成工单号
3. 工单查询可返回正确状态与最近更新信息
4. 后台可更新工单生命周期并发送回复
5. 关键事件可触发邮件通知
6. 所有 MVP 页面支持中英文切换
7. 工单查询失败场景返回统一提示，且限流策略生效
8. 附件上传支持最多 3 张图片，单张默认不超过 5MB

## 9. 里程碑（2 周 MVP）

- 第 1-2 天：PRD 定稿与页面流程确认
- 第 3-5 天：API 与数据模型实现
- 第 6-8 天：前台页面开发与双语接入
- 第 9-10 天：后台工单处理页面开发
- 第 11-12 天：测试、缺陷修复、UAT 验收
- 第 13-14 天：上线准备与发布

## 10. 风险与应对

- 时间压力大 -> 严格控制 MVP 范围
- 翻译质量不稳定 -> 尽早锁定术语表
- 客服流程不清晰 -> 提前定义状态流转与 SLA
- 邮件/上传稳定性风险 -> 采用成熟第三方服务商
